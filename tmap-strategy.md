# TMAP Portfolio Strategy

## 1. 지원 목표
- 회사: 티맵모빌리티
- 직무: Product Designer / UXUI Designer
- 목표: 기존 "SaaS B2B 중심" 포지셔닝을 "디바이스/현장 UX + SaaS 구조화 + AI/Data UX" 조합으로 수정하여 모빌리티 UX 역량을 증명

## 2. 핵심 포지셔닝
7년 3개월차 Product Designer로서 현장 디바이스(태블릿, 워치) 기반 앱, 복잡한 업무 흐름(SaaS), AI 기능, 데이터/통계 화면을 설계해왔습니다. 
이러한 경험을 바탕으로, 이동 환경에서 빠른 상황 판단이 필요한 모빌리티 서비스에 최적화된 "현장과 시스템 사이의 복잡한 흐름을 단순하게 만드는 디자이너"임을 보여줍니다.

## 3. 채용자가 이해해야 할 점
- 스마트워치, 태블릿 등 디스플레이 제약이 강한 환경에서 시인성 높은 UX 설계 경험이 있다.
- 모빌리티 맥락과 일치하는 블루투스 페어링, 하드웨어 센서 데이터 시각화 역량이 있다.
- 복잡한 B2B Task Flow를 구조화하고 데이터 기반 판단을 돕는 UI를 설계할 수 있다.
- 단순 SaaS 화면 제작자가 아니라, 디바이스/현장 UX를 먼저 이해하는 실무형 시니어 디자이너다.

## 4. 추천 프로젝트 순서
1. Caddie App Tablet UX (현장/태블릿 환경 조작 및 판단 UX)
2. Voice Caddie Device UX (워치 GUI, 모바일 연동 및 통계 시각화)
3. Happytalk Task Flow UX (복잡한 시스템 논리 구조화)
4. Dashboard / Statistics UX (데이터 기반 의사결정 지원)
* AI Knowledge Base UX는 후보 프로젝트로 보존.

## 5. 포트폴리오 웹 구현 전략
- **Hero & Summary**: 디바이스/현장 UX 중심으로 전진 배치하여 TMAP 직무 요건과의 연결고리를 명확히 함.
- **Data-Driven**: 실제 검증된 이력서 기간과 사실만 기재. 없는 성과는 만들지 않음.
- **Tone & Manner**: 기계적인 기획서 느낌을 탈피하고, 불필요한 한자어와 추상적 단어를 완전히 배제하여 초등학생도 이해할 수 있는 직관적인 Toss 톤앤매너 유지.
