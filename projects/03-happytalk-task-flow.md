# P03: Happytalk Task Flow UX

## 메타 데이터
- 기간: 2024.07 - 현재
- 회사: Blumn AI (구 엠비아이솔루션)
- 역할: Product Design
- 플랫폼: Web SaaS
- 타겟 유저: 상담 운영자, 상담사, 관리자

## 1. 프로젝트 요약
상담 운영 흐름과 SaaS 핵심 기능을 사용자 행동 단위로 재정리한 프로젝트입니다.

## 2. 티맵 연관성 (Why TMAP)
디바이스 중심의 UX 역량에 더하여, 복잡한 시스템 내부의 업무 흐름(Task Flow)을 논리적으로 구조화하는 능력을 증명합니다. 실시간 상황에서 사용자의 다음 행동을 명확하게 이끌어낸 경험입니다.

## 3. 핵심 문제 (Problem)
상담 운영 과정에서는 여러 기능과 상태가 연결되어 사용자가 다음 행동을 빠르게 판단하기 어려운 구간이 있었습니다. 단순 화면 개선이 아닌, 복잡한 B2B SaaS의 흐름 자체를 정리해야 했습니다.

## 4. 해결 전략 (Strategy)
업무 흐름을 Task 단위로 재정리하고, 사용자의 판단 지점을 기준으로 화면과 플로우를 구조화했습니다.

## 5. 나의 기여 (Ownership)
해피톡/헤이데어 SaaS UX 개선, 상담 UX, 유저 에디터, 온보딩, 어드민 관제 페이지 등 핵심 기능 UX 설계와 디자인 시스템/협업 프로세스 정비를 리딩했습니다.
