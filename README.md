# Portfolio — TMAP Mobility Product Designer

A markdown-based portfolio system for applying to TMAP Mobility as a Product Designer.

## Candidate Profile
- **Experience**: 8 years, SaaS B2B UX/UI Design
- **Target Role**: Mobility Product Designer, TMAP
- **Positioning**: Complex B2B systems → Mobility product design at scale

## Structure

```
portfolio-tmap/
├─ README.md                          ← You are here
├─ portfolio-master.md                ← Portfolio blueprint & content plan
├─ resume-master.md                   ← Resume source of truth
├─ tmap-strategy.md                   ← Narrative strategy for TMAP
├─ reference-design.md                ← Visual/design references & inspiration
├─ company-requirements/
│  └─ tmap.md                         ← JD analysis & company research
├─ projects/
│  ├─ 01-caddie-tablet.md             ← Golfzon Caddie Tablet UX
│  ├─ 02-happytalk-task-flow.md       ← Happytalk Task Flow Optimization
│  ├─ 03-ai-knowledge-base.md        ← AI Knowledge Base Feature
│  └─ 04-dashboard-statistics.md      ← Dashboard & Statistics Design
├─ assets/
│  ├─ figma-links.md                  ← Figma file links & access info
│  └─ screenshots/                    ← Exported screens & visuals
└─ outputs/
   └─ tmap-portfolio-outline.md       ← Final portfolio outline for submission
```

## Workflow
1. Fill `company-requirements/tmap.md` with JD analysis
2. Define narrative in `tmap-strategy.md`
3. Write each project in `projects/`
4. Consolidate in `portfolio-master.md`
5. Export final outline to `outputs/tmap-portfolio-outline.md`

## Rules
- **All design follows [Apple.md](./Apple.md)** — typography, color, layout, components
- Placeholders where evidence is missing — never invent metrics
- Keep content recruiter-scannable (bullet > paragraph)
- Every project connects back to TMAP relevance
